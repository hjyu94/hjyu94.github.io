Little Piccolos — Restaurant Ordering & POS Platform 2025-05 ~ Present · 2026-02-15

🌐 Live Site | 📄 API Docs

Live in production · littlepiccolos.com
Integrated ordering, payment, and kitchen print platform for a Melbourne restaurant. Planned expansion to SaaS subscription for Australian market.

프로젝트 개요

Full-stack platform integrating restaurant online ordering, payment, order management, and kitchen printing. Currently running in production at Little Piccolos and planned to expand as a SaaS platform for restaurants in Australia.

기술 스택

분야 기술
Frontend Next.js, TypeScript, Tailwind CSS
Backend NestJS, TypeORM, PostgreSQL, JWT Authentication, Google OAuth2.0, Role-based Authorization
Printer Python polling script, ESC/POS thermal printer
Infrastructure AWS EC2, AWS RDS, AWS S3, AWS SNS

핵심 기능

  • Menu browsing with category-based navigation
  • Menu customization system with toppings and options
  • Stripe-based online payment processing
  • Support for pickup, dine-in, and delivery orders
  • Pre-order scheduling system
  • Tablet-based POS system for staff
  • Order status management and automatic kitchen printing
  • SMS confirmation sent to customers after successful payment
  • Store configuration including opening hours and shop information

운영 경험

  • Designed, deployed, and operate a production system used in a real restaurant
  • Processes approximately 20–50 real customer orders per day
  • Responsible for menu updates and operational configuration
  • Perform deployment and ongoing system maintenance

회원/인증

  • 비밀번호 bcrypt 암호화 저장, 이메일·닉네임 중복 불가
  • 가입 인증 메일, 패스워드 분실 시 임시 비밀번호 이메일 전송
  • 메일 재전송 시 시간 텀으로 무분별 서버 사용 방지
  • JSESSIONID 쿠키, 로그인 유지 시 30분
  • 프로필: 미설정 시 닉네임 기반 아이콘, 설정 시 프로필 사진
  • 동네·관심 카테고리 설정 → 알림(이메일/사이트 내) on/off
  • 탈퇴 시 게시물은 유지

Spring Security

폼 로그인, 인증 필요 경로(게시물 작성, 쪽지, 알림) vs 허용 경로(로그인, 가입, 검색, static) 분리. DelegatingPasswordEncoder(bcrypt). 닉네임 또는 이메일 + 패스워드로 인증. SecurityContextHolder에 인증 정보 저장.

로그인 기억하기: JdbcTokenRepositoryImpl + persistent_logins 테이블(series, username, token, last_used). remember-me 쿠키로 세션 만료 후에도 재로그인.

메일 시스템

EmailService 인터페이스로 추상화. @Profile("local") ConsoleEmailService(콘솔 로그), @Profile("dev") HtmlEmailService(Google SMTP, MimeMessage). Thymeleaf 템플릿으로 HTML 메일. 비동기 전송.

전송 시점: 가입 인증, 패스워드 갱신, 관심 카테고리·동네에 새 글이 등록되었을 때.

주요 기능

  • 판매글: 등록/조회/수정/삭제. 이미지 최대 5개, 원본+썸네일 저장. Form/Ajax 업로드. 판매완료 상태 변경
  • 채팅: 게시물→프로필→메시지 보내기로 채팅방 생성. WebSocket + Ajax로 한 페이지 내 메시지 송수신
  • 알림: 관심 글 등록 시 nav 바에 새 알림 개수 표시 + 이메일. Event Publisher 사용, on/off 가능

Tech: Spring Boot, Spring Security, JPA, PostgreSQL, WebSocket, Thymeleaf, Bootstrap
