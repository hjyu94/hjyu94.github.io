LionKing (C++ & WinAPI, From Scratch) 2019-04-27

📁 GitHub

프로젝트 개요

1994년 세가 제네시스로 출시된 The Lion King의 2D 플랫포머 모작 프로젝트입니다. Unity, Unreal Engine 등 상용 엔진을 사용하지 않고, Win32 API와 C++만으로 게임 루프, 렌더링, 물리, 오디오, 에디터까지 직접 구현했습니다.

기술 스택 요약

영역 기술 비고
플랫폼 Windows (Win32 API) wWinMain, CreateWindowW, PeekMessage
언어 C++ MSVC v140, Win32/x64 빌드
렌더링 GDI (Graphics Device Interface) BitBlt, HDC, CreateCompatibleDC
이미지 BMP 스프라이트 시트 LoadImage, LR_LOADFROMFILE
오디오 FMOD Ex BGM, SFX, 채널별 재생
데이터 바이너리 파일 I/O CreateFile, ReadFile (엔진 없이 커스텀 포맷)

1. 게임 루프 (Fixed Timestep)

메시지 펌프 + 게임 루프 분리. 10ms 고정 타임스텝으로 물리/업데이트 일관성 확보. 엔진 없이 GetTickCount() 기반 타이밍 제어. 

while (msg.message != WM_QUIT) {
    if (PeekMessage(&msg, ...)) {
        TranslateMessage(&msg);
        DispatchMessage(&msg);
    } else {
        if (oldTime + 10 < GetTickCount()) {  // ~100 FPS 고정
            Main.Update();
            Main.LateUpdate();
            Main.Render();
            oldTime = GetTickCount();
        }
    }
}

2. 객체 관리 시스템 (Object Manager)

  • Object ID 기반 분류: OBJID::MONSTER, PLAYER, ITEM, MAP, BLOCK 등
  • 추상 팩토리: CAbstractFactory<T>::Create(fx, fy) - 템플릿 기반 객체 생성
  • 싱글톤 매니저: DECLARE_SINGLETON 매크로로 매니저 클래스 일원화

3. 씬/스테이트 관리 (Scene Manager)

씬 전환: SceneChange(SCENEID)로 현재/다음 씬 관리. 각 CScene 파생 클래스에서 Initialize, Release 구현. 에디터 씬(LineEditor, Stage2Editor)은 런타임과 동일한 데이터 포맷 사용.

4. 렌더링 파이프라인

  • 더블 버퍼링: BackBuffer.bmp → CreateCompatibleDC → BitBlt 플립
  • 스크롤 오프셋: CScrollMgr::m_iScrollX/Y로 카메라 좌표 보정
  • 스프라이트: FRAME 구조체 기반 프레임 애니메이션

5. 충돌 시스템

방식 용도 구현
RECT 충돌 일반 오브젝트 IntersectRect() + Collision_Proc() 콜백
AABB + 반응 플레이어-블록 CheckRect() → fMoveX/fMoveY로 밀어내기
구(Sphere) 특수 이벤트 sqrt(fx² + fy²) <= fRadiusSum
라인 충돌 경사로/발판 LineCollision(fInX, fInY, pOutY) - 선형 보간으로 Y 좌표 계산

6. 라인/경로 시스템 (2D 플랫포머 핵심)

라이언킹 원작의 경사로, 발판, 그네 등을 구현하기 위한 커스텀 시스템. LINEPOS 좌표 2개로 선분 정의, Vertex/TailVertex로 라인 위 이동 포인트. 데이터 드리븐: Line.dat, Vertex.dat 바이너리 로드. LineEditor로 라인 배치 후 저장.

7. 오디오 (FMOD)

채널 분리: BGM, PLAYER, MONSTER, EFFECT, UI. CSoundMgr::Get_Instance()->PlayBGM(), PlaySound(). FMOD는 라이브러리 수준, 게임 로직과 완전 분리.

8. 데이터 파이프라인

Line.dat, Vertex.dat, Block.dat, VerticalBlock.dat, Grass.dat, Item_HP.dat 등 오브젝트별 바이너리 직렬화. 에디터 ↔ 런타임 동일 포맷으로 데이터 공유.

구현된 게임 요소

  • 플레이어: 상태 머신 (IDLE, ROLLING, HURT, LEDGE_GRAB 등 20+ 상태)
  • 적: Hyena, Lizard, Fly, HedgeHog, Giraffe, Monkey, Ostrich, Rhino 등
  • 특수 동작: 그네(Swing), 코끼리 등 타기(Riding), 코뿔소 꼬리 잡기(Dangling)
  • 아이템: HP, MP, LIFE, SAVE
  • 블록: 수평/수직 블록, 라인 기반 경사면

엔진 프로그래머 관점에서의 하이라이트

  1. 게임 루프 직접 구현 - 메시지 펌프와 업데이트 분리, 고정 타임스텝
  2. 매니저 패턴 - ObjMgr, BitmapMgr, SceneMgr, CollisionMgr, ScrollMgr, KeyMgr, SoundMgr, LineMgr
  3. 추상화된 객체 인터페이스 - CObj 기반 Initialize/Update/LateUpdate/Render/Release
  4. 2D 플랫포머 물리 - 라인 기반 지형, AABB 충돌 반응, 점프/중력
  5. 데이터 드리븐 설계 - 바이너리 데이터 + 에디터 툴로 콘텐츠 제작
  6. 메모리 관리 - _CrtSetDbgFlag 디버그 힙, 수동 new/delete

프로젝트 규모

소스 파일 50+ cpp/h, 해상도 650×450, 타일 64×64, 맵 100×100 타일 지원. 씬: Logo, Stage 1, Bridge, Stage 2, Dead, Restart, LineEditor, Stage2Editor.

빌드 환경

Visual Studio 2015 (v140), Windows 8.1 SDK, Debug/Release, Win32/x64. 외부 라이브러리: FMOD Ex (fmodex_vc.lib).

Tech: C++, Win32 API, GDI, FMOD Ex

상용 엔진 없이 게임의 핵심 시스템(루프, 렌더링, 물리, 오디오, 데이터)을 직접 설계·구현한 경험을 보여줍니다.

Talesweaver (C++ & WinAPI, From Scratch) 2019-06-17

📁 GitHub

프로젝트 개요

상용 게임 엔진(Unity, Unreal 등) 없이 네이티브 C++로 구현한 2D RPG 클라이언트 모작 프로젝트입니다.

항목 내용
프로젝트명 Talesweaver 모작
플랫폼 Windows (Win32 API)
해상도 800 x 600
빌드 Visual Studio 2015 (v140), MSBuild
아키텍처 Win32 + DirectX 9 + FMOD

2.1 그래픽 렌더링

기술 버전/사양 용도
DirectX 9 d3d9.lib, d3dx9.lib 2D 렌더링, 스프라이트, 폰트, 라인
D3DXSprite D3DX 라이브러리 스프라이트 배칭, 2D 드로우콜 최적화
D3DXFont Normal12, Bold15, Heavy15 UI 텍스트 렌더링
D3DXLine D3DX 라이브러리 디버그/UI 라인 그리기

구현 포인트: CGraphicDev 싱글톤으로 디바이스/스프라이트/폰트 통합 관리. Render_Begin() / Render_End() 패턴으로 프레임별 렌더 루프. TEXINFO 구조체로 텍스처 + 스프라이트 메타데이터 통합.

2.2 텍스처/리소스 시스템

  • CTextureMgr: 싱글톤 기반 텍스처 캐시
  • SingleTexture / MultiTexture: 단일 이미지, 스프라이트 시트(애니메이션 시퀀스)
  • 저장 구조: map<wstring, CTexture*> - ObjKey/StateKey 기반 조회
  • SPRITE_INFO: 프레임 수, 경로, 프레임 속도, 컬러키, 피벗

2.3 오디오

FMOD Ex (fmodex_vc.lib). 채널 분리: BGM, PLAYER, MONSTER, EFFECT, NPC, UI (6채널). CSoundMgr 싱글톤, map<TCHAR*, FMOD_SOUND*> 사운드 풀.

2.4 게임 루프 및 타이밍

PeekMessage 기반 논블로킹 메시지 펌프. Progress() → Late_Progress() → Render(). CTimeMgr: LARGE_INTEGER + QueryPerformanceCounter 델타타임.

2.5 입력 처리

CKeyMgr 싱글톤, 비트플래그 기반 키 상태. DWORD 비트마스크 (KEY_LBUTTON, KEY_RBUTTON, KEY_1~8, KEY_SHIFT, KEY_F1~F8 등). S_PLAYER, S_NPC, S_UI_MENU, S_INVEN_WND 등 컨텍스트별 입력 분기.

3. 아키텍처 및 디자인 패턴

  • 씬 관리: CSceneMgr - SC_LOGO, SC_MENU, SC_STAGE, SC_TOWN1, SC_STORE0~2, SC_FIELD0, SC_FIELD3
  • 객체 관리: CObj 추상 베이스, CObjMgr OBJID별 리스트, CObjFactory 템플릿 기반 생성
  • 충돌: CCollisionMgr - CollisionRect(AABB), CollisionRectEX, CollisionSphere, Push_Src
  • 경로 탐색: CAstar A* 알고리즘, Open/Close 리스트, 타일 기반
  • 옵저버 패턴: CSubject/CObserver - CDataSubject, UIObserver, InvenObserver
  • 싱글톤: DECLARE_SINGLETON, IMPLEMENT_SINGLETON 매크로

4. 맵/타일 시스템

타일 크기 TILECX 94, TILECY 48 (다이아몬드/이소메트릭). TILE 구조체 (vPos, vSize, TILE_OPTION, TILE_DRAW). TILE_OPTION: TO_MOVE, TO_NO_MOVE. CMapMgr: map<wstring, BACK_INFO*> 맵별 백업 정보.

5. 게임 시스템 구현

CPlayer, CMonster, CNpc, CItem/CItemMgr, CEquipment, CSkill, CInvenWnd, CShopWnd, CPotion, CDamage, CEffect, CGate 등. 플레이어 스탯/경험치, 장비, 스킬, 인벤토리, 상점, 포션, 데미지, 이펙트, 게이트(씬 전환).

6. UI 시스템

WndUI, MyButton, HPBar, MPBar, Gauge, MonsterHPBar, QuickSlot(F1~F8), MenuUI, Chat. CObj::m_vScroll 정적 멤버로 카메라/월드 스크롤.

7. 데이터 및 파일

CFileInfo_Client 텍스트 파일 라인 단위 파싱. 툴에서 생성한 텍스트 기반 설정 (이미지 경로, 맵 데이터 등). 공유 헤더: Include.h, Macro.h, Value.h, Struct.h, Typedef.h, Function.h.

8. 빌드 환경

Win32/x64, v140 (VS2015), Unicode. DirectX $(DXSDK_DIR). 공통 헤더 ../Headers. 전처리기: WIN32, _DEBUG/NDEBUG, _WINDOWS.

포트폴리오 하이라이트 (엔진/게임 프로그래머 관점)

  1. 엔진 없이 DirectX 9로 2D 렌더 파이프라인 직접 구성
  2. 싱글톤 + 팩토리 + 옵저버 패턴 활용한 확장 가능한 구조
  3. A* 경로 탐색 직접 구현
  4. AABB/Sphere 충돌 및 밀어내기 처리
  5. 델타타임 기반 프레임 애니메이션 및 게임 루프
  6. 비트플래그 입력 + 컨텍스트별 키 레이어
  7. 씬/오브젝트 매니저로 씬 전환 및 렌더 순서 제어
  8. FMOD 멀티채널 사운드 시스템
  9. 타일 기반 맵 + 이소메트릭 타일 (94x48)
  10. 툴-클라이언트 분리 구조 (툴은 MFC로 별도 프로젝트)

Talesweaver 툴 (MFC) - 맵/스프라이트 에디터 2019-06-17

📁 GitHub

툴 개요

맵 편집 및 스프라이트(이미지 에셋) 메타데이터 생성 — 클라이언트에서 사용할 데이터를 제작하는 에디터입니다.

항목 내용
프로젝트명Tool96
UI 프레임워크MFC (Microsoft Foundation Classes)
아키텍처Doc/View + Splitter + TabCtrl
그래픽DirectX 9 (맵/스프라이트 미리보기)
빌드Visual Studio 2015 (v140), UseOfMfc: Dynamic

2. 툴 구성 (4개 탭)

  • CMapTab: 맵 편집 (타일 배치, 오브젝트 배치)
  • CSpriteTab: 스프라이트 에셋 메타데이터 편집
  • CUnitTab: 유닛(몬스터/NPC 등) 배치 정보 관리
  • CPathFind: 경로 데이터 / 단일 이미지 경로 관리

CSplitterWnd: 메인 뷰(맵/스프라이트 미리보기) + 폼 뷰(탭 패널) 분할. CTool96View: CScrollView 상속 — 스크롤 가능한 맵/스프라이트 뷰포트.

3. 맵 탭 (CMapTab)

맵 배경 선택, 5종 타일(Tile0~4) 배치, ObjKey/StateKey 기반 오브젝트 배치. 스케일/알파/스케일 조절. CHJ_Map: Tile_Initialize, Load_File, Tile_Change, Create_Object, Picking/PickingDot. 타일 TILECX 94, TILECY 48 (이소메트릭).

4. 스프라이트 탭 (CSpriteTab)

폴더 트리(CTreeCtrl)로 Texture 폴더 탐색. ObjKey/StateKey 시퀀스, 컬러키, 프레임 속도, 역재생, 피벗(중심점) 설정. 타이머 기반 프리뷰 애니메이션. CFileInfo::DirInfoExtraction(): 폴더 재귀 탐색 → ObjKey/StateKey 자동 추출.

5. 유닛 탭 (CUnitTab)

맵 선택, 유닛(몬스터/NPC) 선택 및 배치. LC_MAP, LC_OBJECT, LC_INDEX 컬럼으로 배치된 유닛 표시. UNIT_INFO (m_wstrMap, m_wstrObjKey, m_iTileIndex, m_tStatusData).

6. 경로 찾기 탭 (CPathFind)

SINGLE_IMGPATH 리스트 관리. OnDropFiles — 드래그 앤 드롭으로 폴더/파일 경로 추가.

7. 파일/데이터 시스템

CFileInfo: ConvertRelativePath, DirInfoExtraction (재귀 탐색), DirFileCount. CFileFind 기반. 경로 규칙: 폴더명%d.png 패턴. 텍스트 기반 출력 — 클라이언트 CTextureMgr, CFileInfo_Client에서 파싱.

8. MFC 활용 요소

CDialog/CDialogEx, CFormView, CScrollView, CSplitterWnd, CTabCtrl, CListBox, CComboBox, CTreeCtrl, CSliderCtrl, CStatic, DDX/DDV, CFileFind, CImage.

9. 툴–클라이언트 연동

맵 데이터 → CStage, CHJ_Map. 스프라이트 경로/메타데이터 → CTextureMgr::ReadImgPath. 유닛 배치 → CMapMgr, BACK_INFO. 공유 헤더 Headers/ (Struct.h, Value.h, Typedef.h)로 데이터 형식 통일.

포트폴리오 하이라이트 (툴 프로그래머 관점)

  1. MFC Doc/View + Splitter + Tab 구조로 멀티 탭 에디터 구성
  2. 맵 에디터: 타일/오브젝트 배치, 피킹, 스크롤, 스케일 조절
  3. 스프라이트 에디터: 폴더 트리 탐색, ObjKey/StateKey 자동 추출, 컬러키/피벗/프레임 속도 설정
  4. CFileFind 기반 디렉터리 재귀 탐색 및 경로 자동화
  5. DirectX 9로 맵/스프라이트 실시간 미리보기
  6. 텍스트 기반 데이터 설계 — 버전 관리·수정 용이
  7. 툴–클라이언트 데이터 형식 공유 (Headers 폴더)
  8. 드래그 앤 드롭 파일 추가
  9. 타이머 기반 스프라이트 애니메이션 프리뷰
